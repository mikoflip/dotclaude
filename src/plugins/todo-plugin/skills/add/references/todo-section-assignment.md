# TODO Section Assignment Guidelines
This document provides guidelines for assigning TODO items to priority sections in `TODO.md`.

## Assignment Priority
1. **Urgency keywords** override effort estimates
2. **Effort estimates** determine base priority
3. **Default** to Medium Priority when unclear

## Guidelines
- **Critical Priority**: Security, blocking, or system failure issues. Items preventing core functionality. Example: "Fix authentication vulnerability"
- **High Priority**: Performance issues or significant improvements. Large, Major, or Epic effort. Example: "Optimize database queries"
- **Medium Priority**: Standard improvements and feature additions. Medium effort. Default section for most items. Example: "Add user preferences"
- **Low Priority**: Nice-to-have features and minor improvements. Trivial or Small effort. Example: "Polish button animations"
