Add improvement suggestions $ARGUMENTS as TODO items in `./TODO.md` of current project

## Argument Format
List of integers, ranges, or a combination: `1`, `1,3`, `1-3`, `1,3-5`

## Execution Steps
1. Get current `CLAUDE_SESSION_ID` from `additionalContext`. If not available, print `Session ID not available. Ensure the SessionStart hook is configured.` and *STOP*
2. Bash `${CLAUDE_SKILL_DIR}/scripts/list-prompts.sh -l <CLAUDE_SESSION_ID>`. If exits non-zero, print error verbatim and *STOP*. If last user prompt does not start with `/todo:suggest`, print `No suggestions found. Please run /todo:suggest first.` and *STOP*
3. Parse arguments. If format invalid, print `Invalid argument format. Use: 1,2,3 or 1-3 or 1,3-5 to select specific items, or no arguments for all items.` and *STOP*. Silently skip any indices that exceed the number of available suggestion items.
4. Read `./TODO.md`. If file not found, print `TODO.md not found. Run /todo:init-md first.` and *STOP*
5. Bash `${CLAUDE_SKILL_DIR}/scripts/list-responses.sh -l <CLAUDE_SESSION_ID>`. This returns the last assistant response, which should be the `/todo:suggest` reply containing a numbered list of 5 suggestion items (each with a title, description, motivation, estimated effort, and impacted files). If exits non-zero, print error verbatim and *STOP*. If the output does not contain the suggestion list, print `Suggestions not found. Please re-run /todo:suggest first.` and *STOP*
6. Filter suggestion items by parsed indices (or keep all if no arguments). If no valid items remain, print `No valid items to add.` and *STOP*
7. Read [Section Assignment Guidelines](references/todo-section-assignment.md). For each valid item, determine a suggested priority section based on urgency/impact and effort, then ask: `Add "{ITEM_TITLE}" to which section? (suggested: {SECTION}) [Critical / High / Medium / Low]`. Wait for input. Re-prompt once if response is not one of the four options.
8. Edit `./TODO.md` to append each valid item under its assigned section, formatted with [TODO Item Template](templates/todo-item.md). Add or remove `{RELATIVE_FILE_PATH}` lines to match the actual number of impacted files in each suggestion. If edit fails, print error verbatim and *STOP*
9. Print [TODO Items Added](templates/todo-items-added.md), repeating `- {ITEM_TITLE}` for each added item. Then *STOP*

## Constraints
- Selected items are included in `TODO.md` using *exact* template
- `TODO.md` is modified (*only* added, *none* removed)
- Print `templates/todo-items-added.md` only — no additional output
