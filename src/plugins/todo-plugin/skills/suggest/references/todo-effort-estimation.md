# TODO Item Effort Estimation Guidelines
This document provides guidelines for estimating required effort for TODO items. Effort estimates must be based on complexity and scope of task, and must follow specified time ranges. When a task spans two tiers, assign the higher one.

| Tier    | Range   | Criteria                                                       |
|---------|---------|----------------------------------------------------------------|
| Trivial | < 0.5h  | Single-line change, no logic, no structural impact             |
| Small   | 0.5–2h  | Localized change, one file, no new abstractions                |
| Medium  | 2–4h    | Multi-file changes, new content sections, minor restructuring  |
| Large   | 4–8h    | Cross-cutting changes, new components, structural additions    |
| Major   | 8–20h   | Significant refactoring, new subsystems, substantial features  |
| Epic    | 20h+    | Requires breakdown into sub-tasks before estimation            |
