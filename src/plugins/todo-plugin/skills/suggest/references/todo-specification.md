# TODO Item Specifications
This document outlines specifications for defining TODO items. Items must be:
- **Atomic**: Single, focused, cannot be broken down further
- **Actionable**: Concrete, can immediately be implemented
- **Specific**: Clear, unambiguous

Items must have a **title** (max 15 words), **description** (max 50 words), and **motivation** (max 30 words).

## Examples
This section provides examples of TODO items in 3 different contexts: software development, content writing, and system administration.

### Software Development Example
- **Title**: Create Primary Button Component
- **Description**: Build a reusable button component with primary styling, hover states, and accessibility attributes following atomic design atom principles.
- **Motivation**: Establish foundational UI element for consistent user interactions across application.

### Content Writing Example
- **Title**: Revise Introduction Paragraph Hook
- **Description**: Rewrite opening sentence to immediately engage reader attention using specific anecdote, statistic, or compelling question relevant to thesis.
- **Motivation**: Strengthen reader engagement and establish clear entry point into argumentative framework.

### System Administration Example
- **Title**: Schedule Network Infrastructure Audit
- **Description**: Coordinate with external vendor to assess current network performance, identify bottlenecks, and document hardware replacement recommendations.
- **Motivation**: Proactively address potential system failures and optimize operational efficiency before performance degradation.
